<h1>Computer Networks (CN) Labs 🌐</h1>

Welcome to my Computer Networks laboratory repository! This space contains all the hands-on networking assignments, configurations, and simulations completed during the semester at Riphah International University.

The goal of these labs is to take theoretical networking concepts and apply them in practical, simulated environments. You will find configurations for routers, switches, and servers, along with step-by-step troubleshooting methods to fix broken networks.

<h2>🛠️ Tools Used</h2>

Cisco Packet Tracer: The primary software used for designing, configuring, testing, and simulating all network topologies.

<h3>Lab 08: Multi-Service Network Topology & Fault Diagnosis</h3>

Description: A complete setup of a multi-service network connecting different local networks (LANs) across a wider network (WAN).

Key Configurations:

* Setting up dynamic routing using RIP version 2 so routers can automatically share network paths.
* Securing router console and VTY lines with passwords to block unauthorized access.
* Setting up an FTP Server with specific permissions (Admin, Student, Guest) for file sharing.
* Configuring a Mail Server with SMTP and POP3 services to send and receive emails.

Analysis: A deep dive into the OSI Model—watching exactly how data packages (DNS, FTP, and SMTP packets) travel hop-by-hop from one computer to another.

Troubleshooting: Identifying and fixing a "silent" network failure where an interface on a central router was shut down, cutting off communication.

<h2>🚀 Key Concepts Covered</h2>

Throughout this semester, these labs focus on making sense of how networks actually work behind the scenes. We tackle:
Routing: How routers talk to each other to find the fastest and best path for data to travel.
Server Management: How files (FTP) and emails (SMTP/POP3) are securely stored, sent, and received across different computers.
Network Security: Locking down network devices so only the right people can control them.
Packet Tracing: Slowing down the network to watch data travel step-by-step, making it easier to spot where things go wrong.

<h2>⚙️ How to Use This Repository</h2>

* Clone this repository to your local computer.
* Ensure you have Cisco Packet Tracer installed to view the simulation files.
* Open the .pkt files included in each lab folder to interact directly with the network topologies.
* Review the attached PDF reports for step-by-step execution, terminal commands, and troubleshooting details.
